# Règles SQF (Arma 3)

- Utilise les conventions de nommage Bohemia: camelCase pour les variables, UPPERCASE pour les constantes
- Préfixe les variables privées avec underscore (_variable)
- Utilise params[] pour récupérer les paramètres de fonction
- Gère les cas null/nil avec isNil
- Utilise spawn pour le code asynchrone, call pour le synchrone
- Optimise les boucles avec forEach plutôt que for
- Documente les fonctions avec des headers standards
- Utilise les namespaces appropriés (missionNamespace, uiNamespace)
